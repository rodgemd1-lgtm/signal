## AOP Process Template (Every Agent)

Every agent follows this AOP structure:

```yaml
agent_id: <name>
group: <group-name>
role: <one-line description>
aop:
  pipeline:
    - id: intake
      description: Receive task, load context
      quality_gate: Task is well-defined
    - id: execute
      description: Run primary capability
      quality_gate: Output matches spec
    - id: review
      description: Self-review against quality criteria
      quality_gate: No critical gaps
    - id: deliver
      description: Ship output, update memory
      quality_gate: Output persisted and indexed
  memory:
    layer1: working    # Session context
    layer2: short_term # Daily results
    layer3: long_term  # Knowledge base
    layer4: persistent # Core identity
  skills: []
  sub_team: []
```
